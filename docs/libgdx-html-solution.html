LibGDX HTML5 Asset Loading Solution

This page demonstrates a reliable approach to load images in LibGDX HTML5/GWT projects deployed on GitHub Pages. The solution addresses common issues like case sensitivity, path handling, and extension capitalization.

The Problem

LibGDX HTML5/GWT deployments often encounter image loading issues, especially on GitHub Pages, due to:

  • Case sensitivity differences between file systems and web servers
  • Path handling inconsistencies
  • Extension capitalization (PNG vs png)
  • GWT's asset preloading behavior

The Solution

Our comprehensive solution has multiple components:

  1. Using lowercase filenames consistently
  2. Creating an image manifest to map original filenames to web-friendly versions
  3. Adding a .nojekyll file to disable GitHub Pages processing
  4. Implementing a robust asset loading strategy in LibGDX code

Java Implementation for LibGDX

Here's how to implement this solution in your LibGDX project:

1. Create an Asset Manager Helper

public class WebAssetManager {
    private static final boolean isWeb = Gdx.app.getType() == Application.ApplicationType.WebGL;
    private final AssetManager assetManager;
    private JsonValue manifest;

    public WebAssetManager(AssetManager assetManager) {
        this.assetManager = assetManager;
        if (isWeb) {
            // Load the manifest first
            assetManager.load("images/image-manifest.json", JsonValue.class);
            assetManager.finishLoading();
            manifest = assetManager.get("images/image-manifest.json", JsonValue.class);
        }
    }

    public void loadTexture(String originalPath) {
        String path = originalPath;
        if (isWeb && manifest != null) {
            // Look up the web-friendly path in the manifest
            if (manifest.has(originalPath)) {
                path = manifest.get(originalPath).getString("path");
            } else {
                // Fallback to lowercase as a last resort
                path = originalPath.toLowerCase();
            }
        }
        assetManager.load(path, Texture.class);
    }

    public Texture getTexture(String originalPath) {
        String path = originalPath;
        if (isWeb && manifest != null) {
            // Look up in manifest
            if (manifest.has(originalPath)) {
                path = manifest.get(originalPath).getString("path");
            } else {
                path = originalPath.toLowerCase();
            }
        }
        return assetManager.get(path, Texture.class);
    }
}

2. Using the Asset Manager in Your Game

public class MyGame extends Game {
    private AssetManager assetManager;
    private WebAssetManager webAssetManager;

    @Override
    public void create() {
        assetManager = new AssetManager();
        webAssetManager = new WebAssetManager(assetManager);
        
        // Load assets with the helper
        webAssetManager.loadTexture("images/DigDugRight.PNG");
        webAssetManager.loadTexture("images/PacManLeft.PNG");
        assetManager.finishLoading();
        
        // Use the assets
        Texture digDugTexture = webAssetManager.getTexture("images/DigDugRight.PNG");
        // ...
    }
}

Step-by-Step Deployment Instructions

  1. Run the prepare-libgdx-images.ps1 script to create lowercase versions and the manifest
  2. Ensure you have a .nojekyll file in your docs folder
  3. Update your LibGDX code to use the WebAssetManager helper or simply use lowercase filenames
  4. Deploy your HTML5 build to the docs folder
  5. Commit and push to GitHub
